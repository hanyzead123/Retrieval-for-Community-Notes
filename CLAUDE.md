# 224N Final Project

Community Notes fact-checking retrieval system: scrapes Wikipedia sources cited in Community Notes, chunks them into passages, and retrieves relevant passages using BM25, hybrid (BM25 + dense), hybrid+rerank, or Gemini parametric retrieval.

## File Structure

| File | Purpose |
|------|---------|
| `preprocess.py` | Loads notes TSV, filters to URL-containing rows, scrapes cited URLs with trafilatura, chunks text (200 words, 50 overlap), writes `passages.jsonl` and `notes_filtered.parquet` |
| `retrieve.py` | Builds BM25 index (Pyserini/Lucene) and optionally dense index (all-MiniLM-L6-v2 + FAISS), retrieves top-10 passages per note |
| `rerank.py` | Cross-encoder reranking of hybrid retrieval results |
| `test.py` | NLI scoring with RoBERTa-large-mnli; adds `nli_score` to retrieval results |
| `evaluate.py` | Computes Recall@k and SupportScore@k from results JSONL |
| `run_pipeline.py` | One-command pipeline: preprocess → retrieve → NLI score → evaluate |
| `gemini_retrieve.py` | Gemini (Vertex AI) parametric baseline: asks Gemini for top-k Wikipedia URLs per note; outputs `results/gemini_results.jsonl` |
| `reduce_tsv.py` | Filters a raw Community Notes TSV to rows with Wikipedia links, keeps only key columns |
| `requirements.txt` | Dependencies |

## Running

```bash
# Full pipeline (BM25 or hybrid)
python run_pipeline.py
python run_pipeline.py --modes bm25 hybrid

# Individual steps
python preprocess.py
python retrieve.py --mode bm25
python retrieve.py --mode hybrid
python test.py -i results/hybrid_results.jsonl
python evaluate.py -i results/hybrid_nli_results.jsonl --k 5

# Gemini parametric baseline (Vertex AI)
python gemini_retrieve.py               # full 100-note test set
python gemini_retrieve.py --limit 5     # quick test
python evaluate.py -i results/gemini_results.jsonl --k 5   # Recall@k only

# Utility: reduce a raw notes TSV to Wikipedia-linked rows
python reduce_tsv.py data/notes.tsv -o data/notes.reduced.tsv
```

## Data

| Path | Format | Description |
|------|--------|-------------|
| `data/notes-small.reduced.tsv` | TSV | 100 notes (test set). Columns: `noteId`, `tweetId`, `summary`, `classification`, `trustworthySources` |
| `data/notes-large.reduced.tsv` | TSV | ~180K notes (full set, gitignored) |
| `data/notes_filtered.parquet` | Parquet | Filtered notes used by retrieve.py (generated by preprocess.py) |
| `data/passages/passages.jsonl` | JSONL | Chunked passages. Fields: `id`, `contents`, `note_id`, `source_url` |
| `data/bm25_index/` | Lucene | Pyserini BM25 index over passages |
| `data/dense_embeddings.npy` | NumPy | Passage embeddings (384-dim, all-MiniLM-L6-v2) |
| `data/faiss.index` | FAISS | Flat inner-product index over passage embeddings |
| `data/passage_ids.json` | JSON | Array mapping FAISS index positions to passage IDs |
| `results/bm25_results.jsonl` | JSONL | BM25 retrieval output |
| `results/hybrid_results.jsonl` | JSONL | Hybrid retrieval output |
| `results/gemini_results.jsonl` | JSONL | Gemini parametric retrieval output (fields: `note_id`, `query`, `rank`, `source_url`) |

Retrieval JSONL fields (BM25/hybrid): `note_id`, `query`, `rank`, `passage_id`, `score`, `passage_text`, `source_url`

## Benchmark Results

BM25/Hybrid/Rerank evaluated on 100-note test set. Gemini is a side experiment on a 1,000-note subset (full 14k run ~4.7 hrs at 1.2s/note).

| System | Notes | Recall@5 | SupportScore@5 |
|--------|-------|----------|----------------|
| BM25 | 100 | 0.8586 | 0.6233 |
| Hybrid (BM25 + dense) | 100 | 0.8900 | 0.6238 |
| Hybrid + Rerank | 100 | 0.8700 | 0.6190 |
| Gemini 2.0 Flash (parametric) | 1,000 | 0.3980 | 0.0000 |

Gemini SupportScore is 0 — returns URLs only, no passage text for NLI. URL matching uses canonical Wikipedia slug normalization (strips fragments, query params, percent-encoding, mobile subdomains).

## Gotchas

- **Java required**: Pyserini needs Java 11+. On Apple Silicon, must use an ARM64 JDK (not x86_64). `JAVA_HOME` is set in `~/.zshrc` to Temurin 21.
- **OpenMP/JVM conflict**: FAISS and Pyserini both load OpenMP. Fixed with `OMP_NUM_THREADS=1` and `KMP_DUPLICATE_LIB_OK=TRUE` at top of `retrieve.py`.
- **Query cleaning**: URLs are stripped from note summaries before searching (regex: `https?://[^\s,\]\)\"]+`).
- **Dense index caching**: `build_dense_index()` skips if all three cache files exist. Delete them to force rebuild.
- **Current corpus**: 6,795 passages from 100 notes. Scaling to 180K notes will massively increase passage count and dense encoding time.
- **Gemini uses Vertex AI**: Requires `GOOGLE_CLOUD_PROJECT` in `.env` and `gcloud auth application-default login`. AI Studio API keys hit quota=0 on free tier. Uses `google-genai` SDK (not the deprecated `google-generativeai`). The `.env` file is gitignored.
